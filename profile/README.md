## ◈ THE QUESTION

> *만약 가야금이 boom-bap의 베이스라인 위에서 인사말이 아니라 **멜로디**를 잡으면 어떻게 되는가?*
>
> *만약 해금이 ambient 패드 안에서 양념이 아니라 **주인공**이면 어떻게 되는가?*

대부분의 fusion은 한국 전통악기를 **맛**으로 쓴다.

1초 인트로의 산조 가락. 후렴 중간에 잠깐 들어오는 장구 한 줄. *한국이 세련되게 다가갈 수 있다*는 것을 보여주는 짧은 인사.

라벨 큐는 그 위치를 거부한다.

여기서 한국 전통악기는 **주연**이고, 현대 장르는 그 주연이 살 수 있는 **세계**다. 가야금이 boom-bap을 *사용*하는 것이지, boom-bap이 가야금을 *빌려 쓰는* 것이 아니다.

이 차이는 작아 보이지만 모든 미세 결정을 바꾼다 — 어느 악기가 멜로디를 가지는지, 어느 악기가 박자를 정의하는지, mix에서 어느 소리가 가장 크게 들리는지, 후렴이 어느 음계 위에 서는지.

한국 전통악기는 무형문화재 박물관에 있는 유물이 아니다. 수백 년의 음악 어휘를 가지고 살아 있는 *연주악기*다. 라벨 큐는 그 사실에서 출발한다.

<br/>

## ◈ ⊗

라벨 큐의 모든 트랙은 두 요소의 **텐서 곱(tensor product)** 으로 표기된다.

```
가야금 ⊗ boom-bap
해금   ⊗ ambient
단소   ⊗ lo-fi
```

⊗는 단순한 더하기가 아니다.

```
A + B   →   두 개를 나란히 놓는 것
A ⊗ B   →   한 차원이 다른 차원을 곱해 새로운 공간이 열리는 것
```

가야금이 boom-bap의 그루브 위에서 *증폭*되고, boom-bap이 가야금의 음계 안에서 *재정의*된다. 두 요소가 자기 정체성을 잃지 않은 채로, **세 번째 무엇**이 된다.

이것이 라벨 큐가 ⊗를 brand mark로 쓰는 이유다. 모든 발매물 표지에, 모든 트랙 페이지에, 이 로고가 적힌다.

<br/>

## ◈ SIX × SIX

|    | 모던 장르          | 한국 악기              | 텍스처                  |
|:--:|:------------------|:----------------------|:-----------------------|
| 01 | **boom-bap**      | 장구 _janggu_          | percussive · low-mid   |
| 02 | **funky**         | 북 _buk_               | thump · energy         |
| 03 | **jazz**          | 가야금 _gayageum_      | plucked melody         |
| 04 | **ambient**       | 해금 _haegeum_         | bowed sustain          |
| 05 | **lo-fi**         | 단소 _danso_           | breath · blur          |
| 06 | **experimental**  | 거문고 _geomungo_      | weight · dissonance    |

이론적으로 6 × 6 = **36개의 조합**이 가능하다. 모두 발행되지는 않는다. 살아남는 것만 발행된다.

> **살아남는다는 것** = 두 요소 모두 자기를 잃지 않으면서, 들었을 때 **하나의 곡으로** 들리는 것.

가장 어려운 조합은 보통 가까운 조합이다. *해금 ⊗ ambient*는 너무 잘 어울려서 진부해지기 쉽고, *장구 ⊗ boom-bap*은 둘 다 percussion이라 뻔해지기 쉽다. 거리가 먼 조합 — *단소 ⊗ funky*, *거문고 ⊗ lo-fi* — 이 가장 가능성이 크다.

## ◈ HOW IT WORKS

```
1. GENERATE   →  iq-composer (Suno API 기반) 가 한 트랙당 30~50개 후보 생성
2. CURATE     →  IQ가 듣고, 살아남는 1개만 선택. 통과율 ~3%
3. ANNOTATE   →  vibe notes를 손으로 작성. 왜 이 트랙이 살아남았는지
4. RELEASE    →  CC-BY-NC-4.0으로 발행. 비상업적 사용 자유
```

이 워크플로우가 라벨 큐를 *공장*이 아닌 **레이블**로 만든다. 양산이 아니라 큐레이션이 본질이다.

> 한 사람 + AI가 운영하는 인디 레이블의 새로운 형태.
> 비용은 0에 가깝고, 심사관은 한 명이며, 모든 결정은 *왜?* 에 답할 수 있어야 한다.

생성 도구 **iq-composer**는 [iq-agent-lab](https://github.com/iq-agent-lab) 산하에서 개발된다. 한 사람이 6개 도메인의 콘텐츠를 양산·큐레이션할 수 있게 하는 더 큰 시스템(iq-blogger, iq-label, iq-writer, iq-painter, iq-studio, iq-curator)의 두 번째 모듈.
